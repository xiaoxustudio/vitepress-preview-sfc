# i18n / Locale

The ViewSfc component supports built-in internationalization with `en` and `zh` presets.

## Auto-detection

The locale is automatically detected from `<html lang>` attribute. In VitePress, this is set by your [locales config](https://vitepress.dev/reference/site-config#locales).

```
// When <html lang="zh-CN"> → uses zh
// When <html lang="en"> → uses en
// Fallback → en
```

## Switching locale programmatically

```ts
import {
	ViewSfcConfigFn,
	ViewSfcTagSymbol
} from "@vitepress-preview-sfc/components";
import { provide } from "vue";

const config = ViewSfcConfigFn();
provide(ViewSfcTagSymbol, config);

// Switch to Chinese
config.setLocale("zh");

// Switch back to English
config.setLocale("en");
```

## Registering custom locales

```ts
const config = ViewSfcConfigFn();

config.locales.jp = {
	collapseText: "折りたたむ",
	copyTextSuccess: "コピー成功",
	copyTextError: "コピー失敗",
	accessibility: {
		btnGroupLabel: "コードプレビューコントロール",
		codeRegionLabel: "ソースコードビューア",
		collapseBtnLabel: "コードを折りたたむ"
	}
};

config.setLocale("jp");
```

## API

| Property            | Type                             | Description                   |
| ------------------- | -------------------------------- | ----------------------------- |
| `locale`            | `Ref<string>`                    | Current locale name           |
| `locales`           | `Record<string, LocaleMessages>` | Registered locale messages    |
| `setLocale(locale)` | `(name: string) => void`         | Switch to a registered locale |
| `collapseText`      | `Ref<string>`                    | Text for collapse button      |
| `showCodeText`      | `Ref<string>`                    | Text for show code button     |
| `copyCodeText`      | `Ref<string>`                    | Tooltip text for copy button  |
| `copyTextSuccess`   | `Ref<string>`                    | Success toast message         |
| `copyTextError`     | `Ref<string>`                    | Error toast message           |
